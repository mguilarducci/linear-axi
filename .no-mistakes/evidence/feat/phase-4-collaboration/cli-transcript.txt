# Phase 4 — Collaboration commands wired into linear-axi CLI

Commit 7e595b6 wires the label, user, document, and initiative commands
into the CLI and refreshes the skill. Below is the end-user terminal
experience, captured two ways:

  A. The REAL built binary (node dist/bin/linear-axi.js) for the wiring,
     help, and auth-error surfaces that need no API key.
  B. The REAL command pipeline (arg parsing -> dispatch -> linearRequest ->
     TOON rendering) driven against a stubbed Linear GraphQL endpoint, so
     the actual rendered list/view/create output is visible.

========================================================================
A. REAL BINARY — wiring & help surfaces (no API key)
========================================================================

$ linear-axi --help
usage: linear-axi [command] [args] [flags]
commands[13]:
  (none)=dashboard, me, team, issue, project, cycle, milestone, label, user, document, initiative, api, setup
flags[2]:
  --help, -v/-V/--version
examples:
  linear-axi
  linear-axi issue list
  linear-axi issue view ENG-123
  linear-axi project list
  linear-axi setup hooks

$ linear-axi label --help
usage: linear-axi label <list|create> [flags]
  list [--team <KEY>]               list labels (name, color, scope)
  create --name "..." [flags]       create a label
    --color <hex>                     label color, e.g. #ff8800
    --team <KEY>                      team-scope the label (else workspace)
$ linear-axi user --help
usage: linear-axi user <list|view> [args]
  list                       list workspace members
  view <EMAIL|NAME|ID>       show a user and their open assigned issues
$ linear-axi document --help
usage: linear-axi document <list|view> [args]
  list                       list documents (title, project, updated)
  view <ID> [--full]         show a document's content
$ linear-axi initiative --help
usage: linear-axi initiative <list|view> [args]
  list                       list initiatives (name, status)
  view <NAME|ID> [--full]    show an initiative and its projects
------------------------------------------------------------------------
Dispatch reaches each new command (structured AUTH_REQUIRED, not
'unknown command') — proving they are registered, not just defined:

$ linear-axi label list
error: LINEAR_API_KEY is not set
code: AUTH_REQUIRED
help[2]: "Set LINEAR_API_KEY to a Linear personal API key, then retry",Create one at Linear → Settings → Security & access → Personal API keys

$ linear-axi initiative list
error: LINEAR_API_KEY is not set
code: AUTH_REQUIRED
help[2]: "Set LINEAR_API_KEY to a Linear personal API key, then retry",Create one at Linear → Settings → Security & access → Personal API keys
