# Contributing

Thanks for wanting to contribute.

## Workflow

1. Fork the repo and create a branch for your change.
2. Make your changes following the repo conventions below.
3. Run `pnpm run build`, `pnpm run lint`, and `pnpm test` before pushing.
4. Open a pull request against `main`.

PRs use [Conventional Commits](https://www.conventionalcommits.org/) for their titles so [release-please](https://github.com/googleapis/release-please) can derive the next version and changelog automatically (`feat:`, `fix:`, `docs:`, `chore:`, …).

## Repo conventions

- Node 20+, TypeScript, ESM-only.
- Install dependencies with `pnpm install --frozen-lockfile`.
- Format with `prettier` and lint with `eslint`; CI runs both.
- Do not hand-edit `CHANGELOG.md` or `.release-please-manifest.json`.
  They are regenerated by release-please from your conventional commit messages.
- Do not hand-edit `skills/linear-axi/SKILL.md`.
  It is generated from the shared CLI guidance in `src/skill.ts`; run `pnpm run build:skill` and commit the result.
  A CI check (`Guard generated files`) fails if any of these are edited by hand.

## Design

This CLI follows [AXI](https://github.com/kunchenguid/axi) (Agent eXperience Interface) principles: token-efficient TOON output, minimal default schemas, structured errors on stdout, definitive empty states, and contextual next-step hints. Keep new commands consistent with those principles — see existing commands and `src/toon.ts` for the output helpers.
